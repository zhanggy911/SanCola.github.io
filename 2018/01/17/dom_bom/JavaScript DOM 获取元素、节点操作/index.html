JavaScript DOM 获取元素、节点操作

发表于 2018-01-17 | 分类于 JavaScript

基本概念

  • DOM定义
    • Document Object Model 文档对象模型
    • 就是把 HTML 文档模型化，当作对象来处理
  • 内容概念
    • Document（文档）
      • 就是指 HTML 或者 XML 文件
    • Node（节点）
      • HTML文档中的所有内容都可以称之为节点，常见的节点有
        • 元素节点
        • 属性节点
        • 文本节点
        • 注释节点
    • Element（元素）
      • HTML 文档中的标签可以称为元素
  • 结构概念
    • 父节点：当前节点的父级
    • 子节点：当前节点的子级
    • 兄弟节点：和当前节点同属一个父级的节点

获取元素

  • getElementById
    • 通过 id 寻找一个元素。该方法只能被 document 对象调用
  • getElementsByTagName
    • 通过标签名寻找一类元素，找到的是由元素对象组成的伪数组
    • 即可以被 document 调用，又可以被元素对象调用，被元素对象调用时表示在该元素对象内部执行查找
  • getElementsByClassName
    • 通过类名获取元素，找到的是由元素对象组成的伪数组，允许元素对象调用
  • querySelector
    • 返回文档中的一个标签，选择时可以使用 CSS 选择器指定，允许元素对象调用
  • querySelectorAll
    • 通过 CSS 选择器获取指定的元素，返回的是一个由元素对象组成的伪数组，允许元素对象调用
  • document.documentElement
    • 获取html标签

节点、元素操作

  • 节点类型
    • 语法：node.nodeType
    • 系统使用数字表示节点含义
      • 1：元素
      • 2：属性
      • 3：文本
      • 8：注释
  • 节点层级
    • childNodes — 子节点
    • children — 子元素
    • nextSibling — 下一个兄弟节点
    • nextElementSibling — 下一个兄弟元素（兼容问题）
    • previousSibling —上一个兄弟节点
    • previousElementSibling — 上一个兄弟元素（兼容问题）
    • firstChild — 第一个子节点
    • firstElementChild — 第一个子元素（兼容问题）
    • lastChild — 最后一个子节点
    • lastElementChild — 最后一个子元素（兼容问题）
    • parentNode — 父节点（一定是元素节点）

节点操作

  • 插入节点
    • insertBefore
      • 语法：父节点.insertBefore(被插入节点)
      • 插入的最终位置在指定的被插入节点之前
      • 此方法会把插入的元素返回
    • appendChild
      • 语法：父节点.appendChild(要插入的节点)
      • 插入的最终位置在父节点的最后
      • 如果是克隆的元素，不影响原来的元素，如果不是，会直接移动位置
      • 此方法会把插入的元素返回
  • 创建元素
    • createElement()
      • 语法：document.createElement('标签字符串')
      • 创建完成后需要使用append方法追加到要使用的位置
  • 克隆节点
    • clone()
      • 语法：被克隆的节点.cloneNode(布尔值)
      • 值为true，深度克隆，值为false，浅层克隆
  • 删除节点
    • removeChild()
      • 语法：父节点.removeChild(被删除的节点)
    • father.innerHTML=""
      • 在清空内容的时候和事件相关的函数不会被销毁，任然遗留在内存中
      • 因此建议使用 father.removeChild(son) 来清除
  • 替换节点
    • replaceChild()
      • 语法：父节点. replaceChild(替换节点，被替换节点)

动态创建结构

  • 方式一：直接在文档中书写
    • document.write("内容")
    • 会清空之前已经有的元素
  • 方式二：改变元素对象内部的HTML
    • innerHTML="内容"
  • 方式三：创建或克隆节点并追加
    • createElement()
      • 需要注意重复添加的问题，使用if语句判断，注意写活
    • cloneNode()
    • 通过这两种方法创建出来的元素只是保存在内存中，必须放到页面上才行
    • 创建完成后必须使用 append 方法追加到要使用的位置

执行效率优化

动态生成页面时，如果遇到需要大量拼接字符串的情况，最好先把字符串放到数组中，再使用 join() 方法加入
