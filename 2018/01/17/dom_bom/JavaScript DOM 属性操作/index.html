JavaScript DOM 属性操作

发表于 2018-01-17 | 分类于 JavaScript

DOM 对象的属性和HTML的标签属性几乎是一致的，常用的有 src、title、className、href 和 style，其中className 对应就是标签的 class 属性，但是由于 class 在 JS 中是关键字，所以变成了className

获取 / 设置属性

  • 基本方法
    • 获取
      • 元素对象.属性名
    • 设置
      • 元素对象.属性名 = “属性值”;
  • 获取属性值：getComputedStyle
    • 在元素应用完有效样式且计算完成所有属性的值后，获取 CSS 值并返回
    • 返回的数值是只读的，带单位
    • 可以从伪元素获取 CSS 属性
    • 存在兼容性问题

常用属性

  • src：引用位置
  • title：名称
  • href：引用位置
  • className：类名
  • style：样式

表单属性

  • type：设置 input 元素的类型（字符串）
  • value：设置 input 元素的值（字符串）
  • checked：设置 input 元素是否选中（boolean值）
  • selected：设置下拉列表 select 中的 option 是否被选中（boolean值）
  • disabled：设置 input 元素是否被禁用（boolean值）
  • setCustomValidity：自定义表单验证

样式属性

  • 概念
    • style 属性是对象，style 对象的属性是字符串
    • style 只能获取和设置行内样式
    • DOM 对象中 style 的属性和标签中 style 内的值几乎一样
      • 带有 - 的属性名在JS中不能识别需要转换，形如 background-color -> backgroundColor
  • 设置和获取样式
    • 通过对象的 style 属性只能设置和获取行内样式
    • 获取样式的时候，如果行内没有该样式，输出空字符串
    • 设置样式的时候，赋的值是一个字符串，如果这个属性有单位，一定要加单位
  • class 控制样式 与 style 控制样式
    • 不严格的说
      • 如果样式很多 通过 直接控制类名 的方式加样式
      • 如果样式比较少 通过直接设置比较方便
    • 严格来说
      • 从标准的角度讲，即使是样式比较少的时候，设置样式也要用类名
      • 但是有一类情况，是无论如何也必须直接通过JS加的，那就是涉及到计算的时候
  • classList 元素类名操作
    • classList.add() — 添加类名
    • classList.remove() — 移除类名
    • classList.toggle() — 切换类名
    • classList.contains() — 查询类名

文本属性

  • innerHTML
    • 获取和设置标签中的内容，设置的内容会当作节点对象被解析到DOM树上，可以写标签
  • innerText
    • 获取和设置标签中的内容，设置的内容会被当作普通文本（有兼容性问题，旧版ff不支持），不可以写标签
  • textContent
    • 获取和设置标签中的内容，设置的内容会被当作普通文本（有兼容性问题，旧版IE（9以下）不支持）

图形属性

offset 系列

  • offsetHeight — 元素实际高度
    • offseHeight = height + padding + border
    • offsetHeight 与 style.height的区别
      • style.height 只能获取行内样式
      • style.height 是字符串，带单位，offsetHeight 是数值，不带单位
      • style.height 可以设置行内样式，offsetHeight 是只读属性，不能设置
  • offsetWidth — 元素实际宽度
    • 用法与 offsetHeight 一致
  • offsetLeft — 元素左边距（相对于带定位的父元素）
    • 到距离自身最近的（带有定位的）父元素的 左侧 的距离
    • 如果所有父级都没有定位则以 body 为准
    • offsetLeft 是自身 border 左侧到父级 padding 左侧的距离
    • offsetLeft 是就近取整的
    • offsetLeft 与 style.left的区别
      • style.left 只能获取行内样式
      • offsetLeft 只读，style.left 可读可写
      • offsetLeft 是数值，style.left 是字符串并且有单位px
      • 如果没有加定位，style.left 获取的数值可能是无效的
      • offsetLeft 以 border 左上角为基准，style.left 以 margin 左上角为基准
  • offsetTop — 元素顶部距离（相对于带定位的父元素）
    • 用法与 offsetRight相同
  • offsetParent
    • 返回该对象距离最近的带有定位的父级
    • 如果当前元素的所有父级元素都没有定位（position为absolute或relative），offsetParent为body
    • 如果当前元素的父级元素中有定位（position为absolute或relative），offsetParent取最近的父级元素

scroll 系列

  • scrollWidth
    • 元素自身除去 border 的宽度（内部可用宽度），包含不可见部分
  • scrollHeight
    • 元素自身除去 border 的高度（内部可用高度），包含不可见部分
  • scrollLeft
    • 页面整体可以滚动时，页面整体被卷去的左侧宽度
  • scrollTop
    • 页面整体可以滚动时，页面整体被卷去的上方高度
  • 滚动坐标存在兼容性问题

client 系列

  • clientWidth
    • 元素自身除去 border 的宽度（内部可用宽度），不包含不可见部分
  • clientWidth
    • 元素自身除去 border 的高度（内部可用高度），不包含不可见部分
  • clientTop
    • 大小等于 borderTop，如果有滚动条会包含滚动条宽度
  • clientLeft
    • 大小等于 borderLeft，如果有滚动条会包含滚动条宽度
  • 页面可视宽度存在兼容问题

自定义属性

  • 自定义属性的概念

  • 自定义属性的方法

    • getAttribute — 获取标签属性
    • setAttribute — 设置标签属性
    • removeAttribute — 移除标签属性

大部分的浏览器规定，HTML 中规定的属性，默认属性操作方式及自定义属性操作方式可以相互设置及获取，但是在使用时还是尽量加以区别

  • 使用 setAttribute 设置的就用 getAttribute 方式获取
  • 使用默认属性操作方式设置的，就使用默认方式获取
