CSS 行高、盒子模型、浮动、定位

发表于 2018-01-17 | 分类于 CSS

行高

  • 浏览器默认字体大小为16px，默认行高18px
  • 行高单位
    • 行高单位为px：行高与文字无关
    • 行高单位不是px：行高＝文字大小*行高值
  • 父元素行高继承
    • 行高单位是px，子元素的行高等于父元素行高
    • 单位是em／百分比，行高＝父元素字体大小*行高值
    • 无行高单位：行高等于子元素大小*行高值
  • 使文字在父元素居中，设置文字行高等于父元素行高

盒子模型

border 属性

  • border-width — 设置边框宽度
  • border-style — 设置边框样式
    • none：无边框
    • solid：实线
    • dotted：点线

允许属性连写，无顺序要求

padding 属性

  • padding — 内边距属性
    • left
    • right
    • top
    • bottom
  • padding、border 属性会影响盒模型的实际大小
    • 实际大小 ＝ 内容宽度＋左右边框＋左右内边距
    • 实际高度 ＝ 内容高度＋ 上下边框＋ 上下内边距
  • 在元素嵌套时，如果子元素没有设置宽度，那么该子元素设置 padding 值的时候，在父元素的宽度范围内子元素宽度不会发生变化

margin 属性

  • margin — 外边距属性
    • left
    • right
    • top
    • bottom
  • 外边距塌陷
    • 当父元素只有内容大小，没有设置浮动，定位等属性，子元素设置了大小，此时给子元素设置margin-top，父元素也会受影响
    • 解决办法：触发元素的bfc机制，如设置 overflow: hidden

margin/padding 属性的连写

  • margin/padding的属性连写方法完全相同，以margin为例

浮动

  • float — 使元素浮动
    • left
    • right
  • 特点
    • 可以使块级元素在同一行内显示
    • 设置浮动后元素不占位置
    • 可实现模式转换（行内元素自动转为块级元素）
    • 设置了浮动的元素会影响后面的其它元素（因为设置浮动使该元素脱离了标准流）
  • 清除浮动
    • 清除浮动就是消除因浮动的影响而造成的父元素塌陷

定位

  • position — 使用定位的方式实现移动元素
    • static — 默认的现实方式
    • absolute — 绝对定位
    • relative — 相对定位
    • fixed — 固定定位
  • position-absolute
    • 元素设置了绝对定位，其父元素没有设置定位，该元素的绝对定位以浏览器左上角为准
    • 元素的父元素设置了其他定位方式（静态定位除外），那该元素以其父元素的左上角为准
    • 绝对定位的元素不占位置（脱标），并可以实现模式转换
  • position-relative
    • 设置了相对定位的元素只能以相对自己的位置设定定位，元素不会脱标，仍然占位置
  • position-fixed
    • 只以浏览器的左上角为基准，元素脱标不占位置，可实现模式转换
  • z-index
    • 使用了定位的元素相互之间具有层级关系
    • 元素设置了定位，那么此元素的层级高于没有设置定位的元素
    • 如果都设置了定位，那需要使用 z-index（数值）来调节层级。
    • 如果父元素设置了定位，那么子元素的层级会跟随父元素。
  • 定位方式的层级关系
    • z-index > position > float > 标准流
    • 后面的会覆盖前面的设置，层级高的覆盖层级低的，子元素层级跟随父元素层级
    • 只要设置了定位，宽度就不继承
