验证码交互流程

  • 页面第一次加载，请求生成验证码的接口，拿到验证码，将验证码绑定到展示的地方
  • 服务端生成验证码的时候，将验证码保存到 Session 中，后续做校验处理的时候要通过 Session
  • 用户在文本框中输入对应的验证码，发请求
  • 通过 Session 读取到刚才保存的验证码和用户通过输入表单提交上来的验证码做比对
  • 如果成功，则成功处理
  • 如果失败，则告诉用户，验证码错误，然后在客户端重新发请求，获取新的验证码
