Node.js 概念、环境配置、基础

发表于 2018-01-18 | 分类于 NodeJS

概念

简单的说 Node.js 就是运行在服务端的 JavaScript
Node开发就是利用 ECMAscript + 第三方开源库 + Node平台环境API进行编程

Node 官网：https://nodejs.org/en/

  • Node.js 是一个构建于 Chrome V8 引擎之上的一个 JavaScript 执行环境
  • Node 采用了 Chrome 的 V8 引擎解析和执行 JavaScript 脚本代码
  • Node 使用 JavaScript 进行编程
  • Node 在基本的 JavaScript 语言之上，为 Ecmascript 提供了大量的底层编程接口
    • 文件IO
    • 网络IO
    • 处理数据
    • 操作数据库
    • 构建网络服务
  • Node.js 特征
    • 事件驱动（event-driven）
      • 在 Node 中，采用了类似于浏览器端的编程思想，事件模型编程思想
    • 非阻塞 IO 模型（non-blocking I/O model）
      • Node 大量使用类似于浏览器中 ajax 的方式进行编程
      • 绝大多数代码都是异步编程模型

环境配置

  • nvm 安装
  • homebrew 安装
  • 直接安装
    • https://nodejs.org/en/download/
  • node 安装目录
    • node 目录
      • /usr/local/bin/node
    • npm 目录
      • 在全局 node_modules 目录中，具体路径为 /usr/local/lib/node_modules/npm
  • Hello World
    • mac下在项目文件夹打开终端输入 node 即可进入 JavaScript 执行环境
    • 允许直接输入代码运行，例如：console.log
    • 允许加载 js 文件直接运行，例如：node 01-hello-world.js
    • control + c + c 退出

Node 基础

路径

文档资源请求路径

  • 包含 html、CSS等文件请求，以及 html 内部的 src、href 等请求
  • 默认设置为以 ‘ / ‘ 开头，即为网站根目录
  • ‘./‘ ，或者没有 ‘/‘ ，表示为相对路径，相对于当前的请求路径
  • 请求路径中可以包含若干值（查询字符串，哈希值等）

文件操作路径

  • 表示后台处理文件时，在服务器上的文件操作路径
  • 一般设置以 ‘ ./ ‘ 开头，，为 node 命令执行的当前目录
  • 以 ‘/‘ 开头的，会直接去文件所在磁盘根目录寻找
  • 不是上两种方式，默认为 ‘./‘ 路径
  • 在读取或写入文件时，可以使用 __dirname 全局变量拼接出绝对路径

Content-Type

  • 使用工具可以查找到每种文件类型的 Content-Type
  • 使用 mime 可以自动识别，文档类型

Node 全局对象

  • Ecmascript
    • javascript 基础语法
  • console
    • 打印输出
  • setInterval(callback, delay[, ...args])
    • 间歇定时器
  • setTimeout(callback, delay[, ...args])
    • 延时定时器
  • setImmediate(callback[, ...args])
    • 立即执行指定函数，异步I/O之后、settimeout() 和 setinterval() 定时器之前调用
  • clearInterval(intervalObject)
    • 清除间歇定时器
  • clearTimeout(timeoutObject)
    • 清除延时定时器
  • clearImmediate(immediateObject)
    • 清除立即执行定时器
  • __dirname
    • 用于获取当前文件所属目录的绝对路径，防止执行node命令所处的目录影响路径的问题
  • __filename
    • 获取当前文件的绝对路径
  • process
    • 用来访问当前运行进程的一些信息
  • global
    • 类似于浏览器中的 window，一个全局对象
    • 在一个模块系统中，所有的模块都共享一个 global
    • 可以通过给 global 挂载成员在多个模块之间全局共享，慎用
  • module
    • 包含当前模块的一些信息
  • exports
    • exports 是 module.exports 接口对象的一个引用
    • 可以把 module.exports.foo=xxx 的形式简写为 exports.foo=xxx
    • 注意：注向外的暴露的接口对象是：module.exports
  • require()
    • 执行被加载模块中的代码
    • 获取被加载模块中的 module.exports 接口对象
  • …

REPL

Node 中类似于浏览器中的 Console 控制台（Read，Eval，Print，Loop）
