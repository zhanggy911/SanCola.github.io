发表于 2018-01-18 | 分类于 NodeJS

全局第三方模块

  • npm社区提供的第三方包，也是通过 require 进行加载，
  • 第三方加载模块有自己的一套加载规则，node 会按照对应的加载机制去加载。

  • 在项目文件夹打开网路服务
  • 安装
  • 使用

nodemon

  • 自动重启 node
  • 安装
  • 使用
    • 项目文件夹打开命令行nodemon .\app.js(文件路径)

项目第三方模块

mime

  • 通过文件路径自动设置 Content-Type

underscore

在线文档 http://underscorejs.org/

  • 可以在 node 环境下使用的类库，为 node扩展了一些工具方法
  • 也可以在浏览器环境下使用
  • 自带模板引擎

ejs

EJS是一个简单高效的模板语言，通过数据和模板，可以生成HTML标记文本。

EJS可以同时运行在客户端和服务器端，客户端安装直接引入文件即可，服务器端用npm包安装

  • 安装 ejs
  • 成员函数
    • Render(str,data,[option]):直接渲染字符串并生成html
      • str：需要解析的字符串模板
      • data：数据
      • option：配置选项
    • Compile(str,[option]):编译字符串得到模板函数
      • str：需要解析的字符串模板
      • option：配置选项
    • 函数 option 配置包含：
      • Cache[boolean]：是否缓存（需要 filename）
      • Filename[string]：缓存名称，用于缓存的 key
      • ContentTex[object]：执行上下文函数
      • compileDebug[boolean]：是否变异 debug，为 true显示堆栈信息
      • Client[boolean]：是否在客户端执行，为 true 返回函数
      • Delimiter[string]：指定分隔符，默认使用%
      • Debug[boolean]：是否设置为 debug 状态，为 true 会在控制台显示生成函数源码
      • _with[boolean]：是否使用 with{} 函数，为 false 数据保存在 locals 变量中
  • 常用标签
    • <% %> 流程控制标签
    • <%= %> 输出标签（原文输出HTML标签）
    • <%- %> 输出标签（HTML会被浏览器解析）
    • <%# %> 注释标签
    • % 对标记进行转义
    • -%> 去掉没用的空格
