    • 创建一个 router 实例

res 方法

  • res.json()
    • 传入一个对象，Express 自动转换为字符串并发送给客户端
  • res.jsonp()
    • 对跨域的数据进行处理
  • res.redirect()
    • 使浏览器重新请求当前路径
    • 对于异步请求来说，服务端的 redirect 是不管用的
    • res.redirect(back)，重定向到请求来源页（通过http请求头的 Referer 确定，找不到 Referer 则为 /）
  • res.render()
    • 配置 ejs 模板引擎后，自动为 res 对象添加 render 方法，用于渲染页面

req方法

  • req.params()
    • 当使用模糊匹配时，req 具备 params() 方法，用于获取路径参数

中间件

用来处理 http 请求的一个具体的环节（可能要执行某个具体的处理函数），一般都是通过修改 req 或者 res 对象来为后续的处理提供便利的使用

在同一个请求处理的所有中间件中，流通的都是同一个 res, req 对象

中间件分类

  • use(function () {req, res, next})
    • 不关心请求方法和请求路径，没有具体路由规则，任何请求都会进入该中间件
  • use('请求路径', function (req, res, next) {})
    • 不关心请求方法，只关心请求路径的中间件
  • get('请求路径', function (req, res, next) {})
    • get请求规则中间件
  • post('请求路径', function (req, res, next) {})
    • post请求规则中间件
  • 注意
    • 在 use 方法中，如果指定了第一个路径参数，则通过 req.path 获取到的是不包含该请求路径的字符串
    • 例如当前请求路径是 /public/a.jpg 则通过 req.path 拿到的就是 a.jpg

形式一

  • 将目录 static 资源暴露出来，可以通过 /static/* 的形式进行访问
  • app.use('/static/', express.static('static目录的绝对路径'))

注意

  • 如果使用了第一个参数，则请求路径必需以该路径开头，否则不处理
    • 第一个参数相当于第二个参数路径的映射，可以使用任何名称

形式二

  • 省略第一个参数，将目录 static 资源暴露出来，不需要任何前缀就可以访问
  • app.use(express.static('static 目录的绝对路径'))

形式三

  • 将目录 static 资源暴露出来，可以通过多种形式访问

获取 id 的几种方式

  • id 可以通过查询字符串进行传递 post /advert/edit?id=xxx 通过 req.query.id 获取id
    • req.query 是通过查询字符串来获取数据
  • id 也可以通过路径进行传递/advert/edit/:advertId 通过 req.params.advertId 获取id
    • req.params 是通过路径参数来获取数据
  • id 也可以通过表单请求体传递 通过 req.body.id 获取id
    • req.body 是通过表单 POST 请求体来获取数据
