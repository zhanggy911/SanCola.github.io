Node 中的模块化

发表于 2018-01-18 | 分类于 NodeJS

模块化概念

模块系统

  • 由很多不同功能的模块文件组成的一个系统就是模块系统
  • node 里模块指的是实现不同功能的 JavaScript 文件
  • 模块之间相互依赖，相互通信，互相引用和调度完成一些列功能
  • 模块与模块之间不能有污染
  • 模块与模块之间通过特定的规则相互通信

require

  • 用来加载一个模块，该方法是同步加载执行的
  • 获取被加载模块中的 module.exports 接口对象
  • 被加载的文件模块后缀名可以省略
  • 模块是一个私有作用域，外部拿不到
  • 用户自己编写的模块，需要通过路径的形式进行加载

exports

  • exports 是 module.exports 接口对象的一个引用
  • 可以把 module.exports.foo=xxx 的形式简写为 exports.foo=xxx
  • 注意：注向外的暴露的接口对象是：module.exports
  • module.exports 是暴露的接口对象
    • 每一个模块中都有一个 module.exports 编程接口对象，默认是一个空对象
    • 可以通过给 module.exports 编程接口对象添加成员向外暴露内部成员

模块加载规则

  • 加载自定义模块规则
    • 以路径形式加载的的文件模块，由开发人员自己编写
  • 加载核心模块规则
    • 核心模块是Node内置的，例如：fs、http、url 等
    • 直接使用核心模块标识名称加载使用
  • 加载第三方模块规则
    • 通过 npm 下载的模块或者包
    • 通过指定第三方模块的标识名称加载
    • 加载顺序
      1. Node 通过判定如果不是文件路径形式的模块，也不是核心模块，node 会基于当前文件模块同级目录中的 node_modules 目录去查找该第三方模块标识对应的目录
      2. 如果在 node_modules 目录中能找到对应的目录，例如 mime，则 Node 去 mime 目录下找 package.json 文件
      3. 如果找到 package.json 文件，则找该文件中的 main 属性，然后根据该属性指向的文件模块进行加载
      4. 如果上述条件某一个环节不成立，则 Node 进入上一级目录中的 node_modules 进行查找，规则同上
      5. 如果一直到当前文件模块所属的磁盘根目录下都没有找到对应的 node_modules 中的第三方模块，则最后报错：can not find module xxx

核心模块

核心模块就是 Node 提供的模块，例如 fs、http、path等。这些模块其实也是文件模块，核心模块一般都是通过一个固定的标识名称进行加载的，也就是说不能写错，固定的。使用什么就加载什么，无论是程序执行效率还是可维护性都非常好。

在线文档地址：https://nodejs.org/dist/latest-v6.x/docs/api/

常用的核心模块及作用：

核心模块名 作用
fs 文件操作
http 网络操作
net 更底层的网络操作
os 操作系统相关
path 文件路径操作
querystring 查询字符串处理
url url操作处理

fs 文件模块

  • 引入 fs 核心对象
  • fs.writeFile()
    • 语法：fs.writeFile(文件路径，要写入的文件数据，回调函数)
    • 如果文件不存在，则直接新建
    • 若果文件存在，直接覆盖
  • fs.readFile()
    • 语法：fs.readFile(文件路径，[内容编码]，回调函数)
  • fs.appendFile()
    • 语法：fs.appendFile(文件路径，写入的文件数据，回调函数)
    • 追加文件，如果文件存在，在文件内部添加信息
    • 如果文件不存在，直接创建

http 网络模块

  • 引入 http 核心模块

  • 创建网络服务对象

  • 为网络对象绑定请求事件

request 对象

  • request.url
    • 请求的路径
    • 该路径包含 ip 地址之后的所有路径信息（包括 ‘/‘）
    • 需要使用 decodeURI() （仅支持 utf8）转化以防止中文乱码
  • request.method
    • 请求方法
  • request.headers
    • 获取当前请求的请求报文头信息
  • request.on(data,callback)
    • 用于接收 POST 方式提交的数据
  • request.on(end,callback)
    • 用于在 POST 方式提交数据结束后执行一个回调处理函数

response 对象

  • response.write()
    • 返回页面信息
    • 写入的数据必须为字符串或二进制数据
  • response.end()
    • 结束请求
    • 允许直接写返回的页面信息，这样就可以直接省略 response.write()
    • 写入的数据必须为字符串或二进制数据
  • response.writeHead()
    • 设置响应头参数（状态码，解析格式等）
    • Content-Type 应该注意使用何种参数
    • 富文本类型不指定编码格式
  • 为网络服务绑定监听事件
  • 基本流程
  • 根据不同请求响应不同页面
  • 合并写法

net 模块

net 模块是一个底层网络操作模块，创建的网络服务不关心协议，可以收发未经处理的数据

url 模块

  • 查询字符串：当表单使用 get 请求时，会把当前表单中的具有 name 的 input 按照下面的方式，拼接成一个“查询字符串”，然后放到url的路径之后，以“?”作为分隔
  • url.parse()
    • 将 url 数据转换为可操作的对象
    • 转换出的对象具备 pathname 属性（请求路径，？前面）
    • 转换出的对象具备 query 属性（查询字符串，？后面）
    • 在 url.parse() 中，使用布尔值可以自动将 query 属性转换为对象

path 模块

  • node 专门用来处理路径的核心模块
  • path.join()

querystring 模块

  • querystring.parse()
    • 用于将一个查询字符串转换为一个对象
