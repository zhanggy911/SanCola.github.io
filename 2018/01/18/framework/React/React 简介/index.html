React 简介（一）

发表于 2018-01-18 | 分类于 framework

React是Facebook开源的一个用于构建用户界面的Javascript库，已经 应用于Facebook及旗下Instagram。和庞大的AngularJS不同，React专注于MVC架构中的V，即视图。 这使得React很容易和开发者已有的开发栈进行融合

  • 文档地址
    • 英文网：https://reactjs.org/
    • 中文网：https://doc.react-china.org/
  • 在线编辑工具
    • http://jsfiddle.net/reactjs/5vjqabv3/
  • React 主要构成
    • 虚拟 DOM
      • 每次需要渲染 DOM 时，会先比较当前内容和待渲染内容的差异，再决定如何更新DOM
    • React 组件
      • 以组件的方式构建UI，将相对独立的模块定义成组件，并最终完成整体UI的构建
    • jsx 语法
      • React写组件采用 JSX 语法，XML 标签会被 JSX 转换工具转换成纯Javascript代码
    • 单向数据流
      • 单向数据流是指只能从一个方向来修改状态

视图概念

定义一个组件时经常同时具备 props、state、event，即组件的属性，状态，和事件

  • Props
    • 属性，解决了view的定义问题，即语义描述
  • State
    • 状态，是view的有穷状态机，根据状态决定组件ui和行为
    • 有穷状态机：http://baike.baidu.com/view/115336.htm
  • Event
    • 事件，是view里元素的行为

jsx 语法

HTML 转义

React 会将所有要显示到 DOM 的字符串转义。如果 JSX 中含有转义后的实体字符，则不会正确显示，因为 React 会自动转义特殊字符。有几种解决办法：

  • 直接使用 UTF-8 字符
  • 使用对应字符的 Unicode 编码
  • 直接插入原始的 HTML

支持的标签和属性

  • 只有官网支持的标签及属性才可以在 jsx 语法中使用
  • 所有属性都采用驼峰命名法，class 属性和 for 属性分别改为 className 和 htmlFor

属性扩散

  • 用于给组件设置多个属性

自闭和标签

所有React组件都支持单闭合以及双闭合

  • 如果单独使用组件，即采用单闭合
  • 如果存在多组件嵌套，即采用双闭

注释

组件外沿用 JavaScript 的注释方式，在一个组件内使用时要采用{/* */}

组件生命周期

组件生命周期状态

  • Mounting：已插入真实 DOM
  • Updating：正在被重新渲染
  • Unmounting：已移出真实 DOM

处理函数

React 为每个状态都提供了处理函数

  • will函数在进入状态之前调用
    • componentWillMount()
    • componentWillUpdate(object nextProps, object nextState)
    • componentWillUnmount()
  • did 函数在进入状态之后调用
    • componentDidMount()
    • componentDidUpdate(object prevProps, object prevState)

React 还提供两种特殊状态的处理函数

  • 已加载组件收到新的参数时调用
    • componentWillReceiveProps(object nextProps)
  • 组件判断是否重新渲染时调用
    • shouldComponentUpdate(object nextProps, object nextState)

DOM 操作

findeDOMNode()

ReactDOM.render 返回的是对组件的引用（对于无状态组件来说返回 null），是一个 ReactElement 对象

当组件加载到页面上之后，可以通过 findDOMNode() 方法获取组件对应的 DOM 元素

注意：findDOMNode() 不能用在无状态组件上。

refs属性

  • 在要引用的 DOM 元素上面设置 ref 属性，然后通过 this.refs.name 来访问对应的 DOM 元素
  • 如果 ref 是设置在原生 HTML 元素上，拿到的就是 DOM 元素，如果设置在自定义组件上，获取到的就是组件实例，这时就需要通过 findDOMNode 来拿到组件的 DOM 元素
  • 因为无状态组件没有实例，所以 ref 不能设置在无状态组件上
  • Refs 是 React 访问到组件内部 DOM 节点的唯一可靠方法
  • Refs 会自动销毁对子组件的引用（当子组件删除时）
  • 不要滥用 refs，它只是在按照传统的方式操作界面：找到 DOM -> 更新 DOM

组件的 DOM 事件监听

表单

状态属性

表单元素具备几种属于状态的属性

  • value，对应 <input> 和 <textarea> 所有
  • checked，对应类型为 checkbox 和 radio 的<input> 所有

  • selected，对应<option> 所有

    注意：

  • 在 HTML 中<textarea>的值可以由子节点（文本）赋值，但是在 React 中要用 value 来设置

  • 表单元素包含以上任意一种状态属性都支持 onChange 事件监听状态值的更改
  • 针对这些状态属性不同的处理策略，表单元素在 React 里面有两种表现形式

受控组件

对于设置了上面提到的对应“状态属性“值的表单元素就是受控表单组件。一个受控的表单组件，它所有状态属性更改涉及 UI 的变更都由 React 来控制（状态属性绑定 UI）。如果希望输入的内容反馈到输入框，就要用 onChange 事件改变状态属性 value 的值

非受控属性

对于没有设置对应的“状态属性“值，那么该表单元素就是非受控表单组件

select 处理

在 React 中统一使用 value 来指定<select>标签的默认选中
