项目初始化

拆分组件

  • 组件拆分经常不能一次性划分好，可以先把大的部分拆分出来，哪里有性能问题再去优化。
  • 模块拆分按照从大往小，从外往内的原则划分
  • 组件化开发，代码组织结构是按 UI 界面划分的，而不是按业务功能区组织
  • 按照生命周期的顺序去看待组件，如果具有多个组件，先看export default 导出的模块
  • 注意 css 和 js 文件的命名规范
  • 注意代码健壮性，业务逻辑完整性

其他

注意

  • setState 方法不是立即赋值，而是在 componentDidUpdate 方法之后才重新为 state 赋值
  • 如果全局为 div 设置了宽高100%，后面组件里面会自动继承全局的，然后效果是撑满全屏，如果只想撑满父组件，需要在 div 中从新设置htight=100%
