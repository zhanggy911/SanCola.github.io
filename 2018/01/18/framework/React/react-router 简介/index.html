react-router 简介

发表于 2018-01-18 | 分类于 framework

React Router 是一个基于 React之上的强大路由库，它可以让你向应用中快速地添加视图和数据流，同时保持页面与 URL 间的同步

路由匹配原理

嵌套关系

  • Router 使用路由嵌套的概念定义 view 的嵌套集合，当一个给定的 URL 被调用时，整个集合中命中的部分都会被渲染
  • Router 会深度优先遍历整个路由配置来寻找一个与给定的 URL 相匹配的路由

路径语法

大部分的路由路径都可以直接按照字面量理解，除了以下几个特殊的符号：

  • :paramName 会匹配一段位于 /、? 或 # 之后的 URL，命中的部分将被作为一个参数
  • () 内部的内容会被认为是可选的
  • * 匹配任意字符，直到命中下一个字符或者整个 URL 的末尾
  • 相对路径
    • 完整的路径由它的所有祖先节点和自身指定的相对路径拼接而成
  • 绝对路径
    • 可以使路由匹配行为忽略嵌套关系。

优先级

路由算法会根据定义的顺序自顶向下匹配路由。需要保证两个兄弟路由节点配置的前一个路由，不能匹配后一个路由中的路径

路由配置

基础配置

  • Router
    • 所有路由均由 Router 组件包裹
  • Route
    • 每一个 Route 组件就是一个路由
  • Link
    • 在页面中使用 Link 标签来跳转页面
  • this.props.children
    • 组件存在嵌套时使用 this.props.children 渲染子组件

添加首页

  • IndexRoute

绝对路径

  • 使用/指定绝对路径，可以减少 URL 层级
  • 使用 Redirect 解决使用了绝对路径造成的路由错误

进入和离开时 Hook

  • 定义 onEnter 和 onLeave 两个 hook ，用于权限验证及数据持久化等

History

React Router 是建立在 history 之上的。 history 监听浏览器地址栏的变化， 并解析这个 URL 转化为 location 对象， 然后 router 使用它匹配到路由，并渲染对应的组件

  • history 的三种形式：
    • browserHistory
    • hashHistory
    • createMemoryHistory
  • 需要引入
  • 使用

browserHistory

使用browserHistory 是推荐的方式

  • browserHistory使用浏览器中的 History API 用于处理 URL
  • 使用了 HTML5 API，对旧浏览器的支持不好
  • 自动优化 URL，会去掉 hash 的 # 号
  • 根目录以下页面的访问不能直接使用 URL，需要服务器支持

hashHistory

使用这种方式可以支持旧版浏览器

  • 不会去掉 hash 的 # 号
  • 所有页面允许直接使用 URL 访问
  • 存在重复执行代码问题，会影响组件生命周期

createHashHistory

用于服务器渲染及前后端同构
