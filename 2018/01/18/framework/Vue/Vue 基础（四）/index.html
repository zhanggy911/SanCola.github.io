发表于 2018-01-18 | 分类于 framework

自定义事件

v-on 绑定事件

  • 使用 $on(eventName) 监听事件
  • 使用 $emit(eventName) 触发事件

绑定原生事件

自定义事件的表单输入组件

  • 允许通过自定义事件来创建自定义表单输入组件，并使用 v-model 实现双向数据绑定
  • 使组件的 v-model 生效，需要在组件中配置

非父子组件通信

  • 兄弟组件之间的通信可以通过 bus 中央事件总线实现
  • 更复杂的数据通信应当使用 vuex

slot 分发内容

为了让组件可以组合，需要一种方式来混合父组件的内容与子组件自己的模板。这个过程被称为内容分发 。Vue.js 使用 <slot> 元素作为原始内容的插槽

单个插槽

  • 子组件模板只有一个没有属性的插槽时，父组件传入的整个内容片段将插入到插槽所在的 DOM 位置，并替换掉插槽标签本身

具名插槽

  • <slot> 可以使用 name 特性来配置如何分发内容。多个插槽可以有不同的名字。具名插槽将匹配内容片段中有对应 slot 特性的元素。
  • 没有 name 特性的插槽被认为是默认插槽，作为找不到匹配的内容片段的备用插槽

作用域插槽

  • 作用域插槽被用作一个 (能被传递数据的) 可重用模板，来代替已经渲染好的元素

动态组件

  • 通过 is 特性，可以使多个组件使用同一个挂载点，并动态切换
  • 使用 keep-alive 指令缓存被切换的组件，可以保留它的状态或避免重新渲染

杂项

  • 编写可复用组件
    • 编写要求
      • 编写组件时考虑好是否进行复用
      • 复用组件应当有清晰的公开接口
    • Vue 组件的 API 来自于 prop、事件和插槽
      • Prop 允许外部环境传递数据给组件
      • 事件 允许从组件内触发外部环境的副作用
      • 插槽 允许外部环境将额外的内容组合在组件中
  • 子组件引用
    • 如果需要直接访问子组件，可以使用 ref 为子组件指定一个引用 ID
    • 应当避免直接访问子组件，尽量使用 prop 和事件
  • 异步组件
  • 高级异步组件
  • 组件约定命名
    • PascalCase（首字母大写） 是最通用的声明约定
    • kebab-case（横线分割命名） 是最通用的使用约定
  • 递归组件
    • 当组件具有 name 选项时，可以在它的模板内部调用自身
    • 使用递归组件必须保证有终止条件
  • 组件循环引用
  • 内联模板
  • X-Template
  • 低开销组件使用 v-once
