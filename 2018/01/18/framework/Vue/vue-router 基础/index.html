vue-router 基础

发表于 2018-01-18 | 分类于 framework
  • 文档：https://router.vuejs.org/

安装

动态路由

  • 可以通过给一个路由添加参数
  • 在 组件 中通过 this.$route.params 获取参数
  • 可以在一个路由中设置多段『路径参数』，对应的值都会设置到 $route.params 中
模式 匹配路径 $route.params
/user/:username /user/evan { username: 'evan' }
/user/:username/post/:post_id /user/evan/post/123 { username: 'evan', post_id: 123 }
  • 除了 $route.params 外，也提供 $route.query（如果 URL 中有查询参数）、$route.hash 等

监听路径参数变化

  • 例如 /user/foo 导航到 user/bar 时，组件实例会被复用，组件的生命周期函数不会再次调用
  • 可以使用 watch 监听 $route 的变化来实现对路由变化做出响应

嵌套路由

  • 如果有嵌套路由，要用children属性去配置，路由路径会自动拼接

用代码进行路由跳转

  • router.push(location, onComplete?, onAbort?)
    • 会向 history 添加一个新的记录，点击浏览器的后退时会回到之前的 URL
    • 点击 <router-link :to="..."> 等同于调用 router.push(...)
  • router.replace(location, onComplete?, onAbort?)
    • 跟 router.push 唯一区别是不会向 history 添加新记录，而是替换掉当前的 history 记录
  • router.go(n)
    • 参数 n 是一个整数，意思是在 history 中向前或者向后多少步

命名路由

  • 可以在创建Router实例的时候，在routes配置中给某个路由设置名称
  • 可以使用 name 属性跳转

命名视图

  • 在跳转一个路由的时候可以同时渲染页面中的多个部分
  • 一个路由渲染多个组件

重定向和别名

  • 重定向
  • 别名

路由组件传参

  • 使用 props 可以将组件和路由结构，提高组件复用性

history 模式

vue-router 具备以下几种导航模式：

  • hash 模式
    • vue-router 默认模式，用于浏览器环境，无需重新加载页面
    • 兼容低版本浏览器，URL 路径有 # 号
    • 可以使用输入 URL 的形式访问页面
  • abstract
    • 用于 node.js 环境
  • history 模式
    • 利用 history.pushState API 来完成 URL 跳转，无需重新加载页面
    • 该模式下 URL 没有 # 号，更美观直接
    • 需要后台支持，否则部分页面无法以直接输入URL 的形式访问
  • history 后台配置注意
    • 要在服务端增加一个覆盖所有情况的候选资源：如果 URL 匹配不到任何静态资源，则应该返回同一个 index.html 页面（将无法返回404页面）
    • 在 vue 内部配置一个覆盖所有路由的情况，返回一个404页面

导航守卫

在 vue 中，参数或查询的改变并不会触发进入/离开的导航守卫

  • 需要通过观察 $watch 对象来应对变化
  • 使用 beforeRouteUpdate

全局前置守卫

  • 使用 router.beforeEach 注册全局前置守卫

全局后置钩子

  • 使用 router.afterEach 注册全局后置钩子

路由独享守卫

  • 使用 beforeEnter 定义

组件内守卫

路由元信息

过渡动效

  • 可以给使用 <transition> 包裹 <router-view> 来为路由切换添加动画效果

单个路由过渡

  • 使每个路由组件有各自的过渡效果，可以为 <transition> 并设置不同的 name

基于路由的动态过渡

  • 可以基于当前路由与目标路由的变化关系，动态设置过渡效果
