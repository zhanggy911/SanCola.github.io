AngularJS 控制器、过滤器

发表于 2018-01-18 | 分类于 framework

控制器 ng-controller

ng-controller 指令用于为你的应用添加控制器，在控制器中，可以编写代码，制作函数和变量，并使用 scope 对象来访问。

  • ng 中的控制器用来对 scope 进行操作
    • 包括初始化数据和定义事件响应函数等
  • ng 用来解耦业务逻辑层和视图层的关键
  • controller 操作 scope，View 则展现 scope 的内容
  • 传统前端程序中大量复杂的 DOM 操作逻辑都被转变成对 scope 的操作

ng-controller 的定义方式

全局定义控制器(1.5 后废弃)

  • 在 全局 定义一个具名的函数
  • 在函数中传递一个参数：$scope, $scope 就是数据模型(固定写法)
    • 在该函数内部实现业务代码
    • 根据视图通过 $scope 初始化模型数据成员
    • 通过 $scope 提供行为函数
    • 将数据模型成员和行为函数绑定到具体的元素上
  • 使用注意
    • 函数名建议以帕斯卡命名法，同时以 Controller 结尾，目的是为了和普通函数做一个区分
    • 函数不需要自己调用，ng 通过 ng-controller 指令将该函数和指定的 HTML 节点元素绑定起来，ng 会自动来执行该函数

挂载在模块下使用

AngularJS 提供模块化解决方法，可以避免控制器函数的全局命名空间污染的问题

  • 通过 angular.module(模块名称，空数组) 创建一个模块
    • 如果值传递了一个参数，表示获取一个已有的模块
    • 如果要定义一个模块，则需要传递一个空数组为第二个参数
    • angular.module(模块名，空数组) 的返回值就是新创建的模块对象
  • 获取模块，调用模块的 controller 方法为该模块创建一个控制器，这个控制器只存在于该模块中

ng-controller 作用域

  • ng-app 相当于根作用域
  • MainController 可以有数个同级的 controller 作用域
  • AController，BController在MainController内部形成单独的作用域，相互不能访问
  • 内部作用域可以访问顶级作用域

filter 过滤器

ng 中提供了过滤器规则，用来方便的格式化数据

内置过滤器

  • currency — 货币处理
  • number — 数字
  • date — 日期
  • json — json格式
  • uppercase — 大写
  • lowercase — 小写
  • orderBy — 排序
  • limitTo — 限制
  • filter — 过滤器里面的过滤器

自定义过滤器

过滤器的本质就是一个方法：你给它一个参数，它帮你返回一个处理过后的数据。

文档：https://docs.angularjs.org/api/ng/type/angular.Module
