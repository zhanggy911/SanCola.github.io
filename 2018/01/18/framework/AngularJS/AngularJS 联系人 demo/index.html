RESTful API

RESTful API 是目前比较成熟的一套互联网应用程序的 API 设计理论，主要目的是为了使前后端分离

阮一峰 - RESTful API 设计指南

  • 协议
    • 使用 HTTPs协议
  • 域名
    • 尽量将 API 部署在专用域名之下
  • 版本
    • 相应 API 的版本号放入 URL
  • 路径
    • 每个网址代表一种资源，网址中不能有动词，只能是名词
    • 名词往往与数据库中的表格名对应，名词使用复数
  • HTTP动词
    • GET — 从服务器取出资源
    • POST — 在服务器新建资源
    • PATCH — 在服务器更新资源（客户端提供改变的属性）
    • DELETE — 从服务器删除资源
    • …
  • 过滤信息
    • API 应当提供参数，过滤返回的结果
  • 状态码
    • 服务器向客户端返回的状态码和提示信息
  • 错误处理
    • 返回的信息中，将 error 作为键名，出错信息作为键值即可
  • 返回结果
  • Hypermedia API

proxy-server 解决跨域问题

为了达到前后端分离的效果，经常要解决跨域问题，目前生产环境下最常用的方式就是使用代理服务器解决

解决跨域问题方式

  1. 使用 JSONP方式（JSONP 只支持 get 请求）
  2. 在响应头加入 Access Control Allow Orgin
  3. 使用 proxy-server 代理服务器

proxy-server 代理服务器

在前端项目中新建代理服务器目录，当最终启动该项目时，由代理服务器的目录启动。这样，前端与后台之间的数据交互完全由代理服务器中转

目录结构

  • contact — 前端项目目录
    • proxy-server
    • js
    • views
    • index.html
    • package.json
    • ….

  • JS 代码中的文件路径永远是相对于该 JS 文件的执行位置

前后端同构

单页面应用中，在客户端刷新页面会向后台发送请求，造成当前页面重新加载，所以一些单页应用会做前后端同构，好处是提升用户体验，有利于 seo，缺点是工作量庞大
