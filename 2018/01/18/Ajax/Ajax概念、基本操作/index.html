Ajax概念、基本操作

发表于 2018-01-18 | 分类于 JavaScript

同步、异步

  • 同步：必须等待前面的任务完成,才能继续后面的任务
  • 异步：不受当前任务的影响
  • 异步更新网站
    • 网站内容更新
      • 常规的网站内容更新，必须通过刷新显示新内容
    • 异步更新
      • 自动加载资源，同时页面不刷新

POST、GET请求方式

  • GET
    • 所有数据都是可以在地址栏中显示，不能用来提交文件
    • 只发送一次提交
    • 浏览器跟服务器会对 url 做出限制，提交的数据量不能过大
    • Ajax 提交值的方式
      • 在 open() 方法处提交
  • POST
    • 默认对于提交的数据量是没有限制的
    • 会发送两次提交
    • 服务器可以对提交的数据进行限制
    • Ajax 提交值的方式
      • 在 send() 方法处
      • 必须设置请求头："contentType","application/x-www-from-urlencoded"

Ajax

概念

  • Ajax：Asynchronous Javascript And XML（异步 JavaScript 和 XML）
  • 核心：JavaScript 的 XMLHttpRequest 对象
  • XMLHttpRequest
    • XMLHttpRequest 是一个浏览器内置对象
    • ajax 使用的依旧是 HTTP 请求,一个完整的 HTTP 请求需要：
      • 请求的网址,方法 get/post
      • 提交请求内容数据,请求主体等
      • 接收响应回来的内容
    • XMLHttpRequest 兼容性
      • 新版本浏览器：var xml=new XMLHttpRequest()
      • IE5 / IE6：var xml=new ActiveXObject("Microsoft.XMLHTTP")

XMLHttpRequest 对象方法、属性

  • open(method,url,async)
    • method：请求的类型，GET 或 POST
    • url：提交到服务器的文件位置
    • async：true / false (异步 / 同步)
      • 如果使用async属性传入flase，那么 onreaderstatechange 事件不会发生
  • send(string)
    • 将请求发送到服务器
    • string 仅用于 POST 请求
  • onreadystatechange
    • 储存函数，每当 readyState 属性改变时调用该函数
  • readyState
    • 存有 XMLHttpRequest 的状态
    • 0：请求未初始化
    • 1：服务器连接已经建立
    • 2：请求已接收
    • 3：请求处理中
    • 4：请求完成，且响应已就绪
  • status
    • 200：‘OK’
    • 404：未找到页面
  • responseText
    • 获得的字符串形式的相应数据
  • responseXML
    • 获得的 XML 格式的响应数据

使用步骤

  • 建立XMLHTTPRequest对象
    • var ajaxObj = new XMLHttpRequest()
  • 使用 open 方法设置和服务器端交互的基本信息
    • 设置提交的网址,数据,post 提交的一些额外内容
    • ajaxObj.open('get','01.ajax_normal.php')
  • 设置发送的数据，开始和服务器端交互
    • send 发送数据
    • ajaxObj.send()
  • 注册回调函数
    • 当服务器回应我们了,我们想要执行什么逻辑
    • 判断状态
  • 更新界面
    • 在注册的回调函数中,获取返回的数据,更新界面

使用 POST 方式传输数据

  • 用法
    • 创建 formData 对象，创建是需要包含获取到的文件；
    • 通过 send 发送
    • 此种方式虽然是通过 post 发送请求，但是需要发送请求头，系统会自动创建
    • php 内的文件操作方法与表单操作的提交文件方法相同；
    • 可以将表单整体作为一个对象通过这种方式一同发送；
