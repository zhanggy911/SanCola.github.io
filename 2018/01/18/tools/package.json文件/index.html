package.json文件

发表于 2018-01-18 | 分类于 tools

package.json 文件一般用来描述项目的一些基本信息，例如入口文件、依赖项、项目介绍、开发作者等数据。

生成 package.json文件

  • 通过 npm init 以向导的形式生成 package.json 文件，也可以加上 -y 参数，一步生成
  • 生成时 npm 会自动寻找 node_modules的依赖项，并填写到 package.json文件中

package.json 属性

  • name
    • 包名
  • version
    • 包的版本号
  • description
    • 包的描述
  • homepage
    • 包的官网 url
  • author
    • 包的作者姓名
  • contributors
    • 包的其他贡献者姓名
  • dependencies
    • 依赖包列表
    • 如果依赖包没有安装，npm 会自动将依赖包安装在 node_module 目录下。
  • repository
    • 包代码存放的地方的类型，可以是 git 或 svn
  • main
    • 模块ID，用于用户导入模块时填写该 id
    • 指定入口文件
    • main 和模块化中的第三方包加载规则有关系
  • script
    • 用于定义脚本命令，如 start,test 等
    • 定义好之后可以直接在命令行下输入定义好的名称，如：npm start
    • 不受支持的字段可以使用 run 命令，如：npm run A
  • keywords
    • 关键字

项目依赖

全局依赖

  • 即在项目运行环境中需要依赖的项目
  • 代码中不能通过 require() 的方式调用全局安装包。全局的安装是供命令行使用的
  • package.json 不保存全局安装的包信息

生产环境依赖

  • 即项目上线后的生产环境依赖
  • 使用--save参数，npm会自动添加包依赖信息到 package.json 文件中的 dependencies 字段中

开发阶段依赖

  • 即项目处于开发阶段时的依赖，生产环境下不一定需要
  • 使用 --save-dev 参数，npm会自动添加包依赖信息到 package.json文件中的 devDependencies 字段中
  • 在进行代码版本管理的时候，无需管理/上传 node_modules
  • 执行 npm install ，npm 会寻找当前项目中的 package.json 文件中的 dependencies 依赖项，然后依次下载所有依赖
