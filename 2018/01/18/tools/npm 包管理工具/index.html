NPM 包管理工具

发表于 2018-01-18 | 分类于 tools

NPM的全称是Node Package Manager，是随同NodeJS一起安装的包管理和分发工具，它很方便的让JavaScript开发者下载、安装、上传以及管理已经安装的包。

安装模块

基础语法

  • npm install 包名称：npm install gulp
  • npm install 包版本号：npm install gulp@3.1.1
  • 默认安装最新版本

安装包信息将加入到dependencies（生产阶段的依赖）

  • S, –save
  • package.json 文件的 dependencies 字段

安装包信息将加入到devDependencies（开发阶段的依赖）

  • D, –save-dev
  • package.json 文件的 devDependencies字段

只安装生产阶段的依赖项

  • production

安装包信息将加入到 optionalDependencies（可选阶段的依赖）

  • O, –save-optional
  • package.json 文件的optionalDependencies字段

精确安装指定模块版本

  • E, –save-exact
  • package.json 文件的 dependencies 字段

npm install命令可以根据 package.json文件 自动安装所有的依赖包

本地安装（local）

全局安装（global）,使用 -g 或 –global

package.json

安装包的信息可保持到项目的package.json文件中，方便后续项目开发或者他人合作使用

  • package.json 属性
    • name — 包名
    • version — 包的版本号
    • description — 包的描述
    • homepage — 包的官网 url
    • author — 包的作者姓名
    • contributors — 包的其他贡献者姓名
    • dependencies — 依赖包列表。如果依赖包没有安装，npm 会自动将依赖包安装在 node_module 目录下。
    • repository — 包代码存放的地方的类型，可以是 git 或 svn，git 可在 Github 上。
    • main — 模块ID，用于用户导入模块时填写该 id
    • keywords — 关键字
