NVM, NPM的版本管理工具

发表于 2018-01-18 | 分类于 tools

卸载已经安装的 node/npm

  • node 安装目录
    • node 目录
      • /usr/local/bin/node
    • npm 目录
      • 在全局 node_modules 目录中，具体路径为 /usr/local/lib/node_modules/npm
  • 卸载当前 node
    • npm ls -g --depth=0
      • 查看已经安装在全局的模块，方便重新安装
    • sudo rm -rf /usr/local/lib/node_modules
      • 删除全局 node_modules
    • sudo rm /usr/local/bin/node
      • 删除 node
    • cd /usr/local/bin && ls -l | grep "../lib/node_modules/" | awk '{print $9}'| xargs rm
      • 删除全局 node 模块注册的软链

安装 nvm

  • 配置环境变量

安装完成后，使用了 node 的插件需要注意重新配置 node 或 node_modules 的路径

  • node
    • /Users/xxxx/.nvm/versions/node/v9.3.0/bin/node
  • node_modules
    • /Users/xxx/.nvm/versions/node/v9.3.0/lib/node_modules

安装切换各版本 node/npm

  • nvm install stable
    • 安装最新稳定版 node
  • nvm install 版本号
    • 安装指定版本
  • nvm use 版本号
    • 切换至指定版本
  • nvm alias default 版本号
    • 设置指定版本为默认

nvmrc配置文件

如果需要解决默认 node 版本与项目版本不一致的问题，可以使用 .nvmrc配置文件解决

  • cd 项目根目录
    • 进入项目根目录
  • echo 4 > .nvmrc
    • 添加 .nvmrc 文件
  • nvm use
    • 自动使用 .nvmrc 文件中配置的版本
  • node -v
    • 检查 node 是否已经切换为对应版本
