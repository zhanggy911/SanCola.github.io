yarn 包管理工具

发表于 2018-01-18 | 分类于 tools

Yarn 是 Facebook, Google, Exponent 和 Tilde 开发的一款新的 JavaScript 包管理工具。目的是解决这些团队使用 npm 面临的一些问题

为什么使用 yarn

安装速度更快

  • 在 npm 中执行包安装任是按包的顺序一个个执行，所以必须等待上一个包被完整安装才会进入下一个
  • Yarn 则并行的执行这些任务，提高了性能。

更清晰的输出

  • 比 npm 更直观的信息提示

CLI 命令

yarn global

  • npm 添加 -g 或 --global 可以进行全局安装，Yarn 使用的是 global 前缀。
  • global 前缀只能用于 yarn add, yarn bin, yarn ls 和 yarn remove，除yarn add外，这些命令都和 npm 等效

yarn install

  • npm install 命令会根据 package.json 安装依赖以及允许你添加新的模块
  • yarn install 仅会按 yarn.lock 或 package.json 里面的依赖顺序来安装模块。

yarn add [–dev]

  • yarn add 添加依赖并将依赖写入 package.json，类似 npm 的 --save 参数；
  • Yarn 的 --dev 参数则是添加开发依赖，类似 npm 的 --save-dev 参数

yarn licenses

  • yarn licenses ls 列出所有已安装包的许可协议
  • yarn licenses generate-disclaimer 生成包含已安装包许可协议的免责声明。某些协议要求使用者必须在项目中包含该协议

yarn why

  • 查找包依赖关系

yarn upgrade

  • 根据符合 package.json 设定的规则而不是 yarn.lock 定义的确切版本来将包更新到最新版本
