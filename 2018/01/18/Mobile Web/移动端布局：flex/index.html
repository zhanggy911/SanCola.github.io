移动端布局：flex

发表于 2018-01-18 | 分类于 Mobile Web

flex简介

弹性盒布局的容器（flex container）指的是采用了弹性盒布局的 DOM 元素，而弹性盒布局的条目（flex item）指的是容器中包含的子 DOM 元素。图中的最外围的边框表示的是容器，而编号 1 和 2 的边框表示的是容器中的条目。

弹性盒布局中有两个互相垂直的坐标轴：主轴（main axis）和 侧轴（cross axis）。

主轴并不固定为水平方向的 X 轴，侧轴也不固定为垂直方向的 Y 轴

  • 通过 CSS 属性声明首先定义主轴的方向（水平或垂直），则侧轴的方向也相应确定下来
  • 容器中的条目可以排列成单行或多行
  • 主轴确定容器中每一行上条目的侧轴排列方向
  • 侧轴确定行本身的排列方向
  • 可以根据不同的页面设计要求来确定合适的主轴方向。有些容器中的条目要求从左到右水平排列，则主轴应该是水平方向的；而另外一些容器中的条目要求从上到下垂直排列，则主轴应该是垂直方向的。

确定主轴和侧轴的方向之后，还需要确定它们各自的排列方向

  • 对于水平方向上的轴，可以从左到右或从右到左来排列
  • 对于垂直方向上的轴，则可以从上到下或从下到上来排列
  • 对于主轴来说，排列条目的起始和结束位置分别称为主轴起始（main start）和主轴结束（main end）
  • 对于侧轴来说，排列行的起始和结束位置分别称为侧轴起始（cross start）和侧轴结束（cross end）
  • 在容器进行布局时，在每一行中会把其中的条目从主轴起始位置开始，依次排列到主轴结束位置；而当容器中存在多行时，会把每一行从侧轴起始位置开始，依次排列到侧轴结束位置。

弹性盒布局中的条目有两个尺寸

  • 主轴尺寸和侧轴尺寸，分别对应其 DOM 元素在主轴和侧轴上的大小
  • 如果主轴是水平方向，则主轴尺寸和侧轴尺寸分别对应于 DOM 元素的宽度和高度
  • 如果主轴是垂直方向，则两个尺寸要反过来
  • 与主轴和侧轴尺寸对应的是主轴尺寸属性和侧轴尺寸属性，指的是 CSS 中的属性 width 或 height。比如，当主轴是水平方向时，主轴尺寸属性是 width，而 width 的值是主轴尺寸的大小。

常用属性

添加到父元素上的属性

  • display:flex — 开启弹性布局
  • flex-direction — 设置主轴方向
    • row — 主轴为水平方向，排列方向与文档方向相同（默认为从左至右）
    • row-reverse — 主轴水平方向，排列方向与文档方向相反
    • column — 主轴垂直方向，从上到下
    • column-reverse — 主轴垂直方向，从下到上

默认情况下，弹性盒容器中的条目会尽量占满容器在主轴方向上的一行。当容器的主轴尺寸小于其所有条目总的主轴尺寸时，会出现条目之间互相重叠或超出容器范围的现象。CSS 属性”flex-wrap”用来声明当容器中条目的尺寸超过主轴尺寸时应采取的行为

  • flex-wrap — 内容尺寸大于主轴尺寸时的行为
    • nowrap — 默认值，可能出现内容元素重叠或者超出容器范围的现象
    • warp — 当内容超出时，开启换行
    • wrap-reverse — 内容超出换行，但是换行后的方向与主轴方向相反
  • flex-flow — 可以将 flex-direction 和 flex-wrap 结合起来
  • justify-content — 设置主轴对齐、排布方式
    • flex-start — 条目在主轴方向左对齐
    • flex-end — 条目在主轴方向右对齐
    • center — 条目在主轴方向居中
    • space-between — 将所有条目在主轴上平铺展开，每个条目之间的距离相等，最左侧条目在主轴左对齐，最右侧条目在主轴右对齐
    • space-around — 将所有条目在主轴上展开，并将主轴平均分成条目的个数，并将条目在分割后的空间居中

  • justify-content — 设置主轴对齐、排布方式
    • flex-start — 条目在主轴方向左对齐
    • flex-end — 条目在主轴方向右对齐
    • center — 条目在主轴方向居中
    • space-between — 将所有条目在主轴上平铺展开，每个条目之间的距离相等，最左侧条目在主轴左对齐，最右侧条目在主轴右对齐
    • space-around — 将所有条目在主轴上展开，并将主轴平均分成条目的个数，并将条目在分割后的空间居中
  • align-items — 设置副轴对齐，排列方式
    • flex-start — 副轴方向左对齐（默认的实际效果其实是上对齐，一下同理）
    • flex-end — 副轴方向右对齐
    • center — 副轴方向上居中对齐
    • baseline — 副轴方向上以内容基线对齐
    • stretch — 如果副轴的计算值是auto，则尽可能是元素在副轴方向上尽可能占满；

添加到子元素上的属性

  • aline-self 设置元素在副轴上的位置，会覆盖父元素上设置的 aline-items 属性
    • flex-start — 左对齐
    • flex-end — 右对齐
    • center — 居中
    • baseline — 基线对齐
    • auto — 与父元素的align-items相同
  • flex — 设置元素在父元素中的占比，允许设置为 rem，百分比，数字等
    • 设置为数字时，无论设置成多少，最终计算的该元素在所有设置了flex的同级元素中的占比

兼容性

弹性布局本身具有多个版本，使用 display：flex 声明弹性布局的规范最早也已经是2011年以后的版本了，所以弹性布局尽量还是使用在移动端的项目中，并且添加浏览器前缀
