移动端常用布局：响应式布局

发表于 2018-01-18 | 分类于 Mobile Web

简介

响应式布局可以为不同终端的用户提供更加舒适的界面和更好的用户体验，而且随着目前大屏幕移动设备的普及，越来越多的设计师采用这个技术。

@meida query

  • 语法
  • 作用
    • 在媒体查询代码中添加判断代码；满足条件是改变外观；
    • 类似于if语句，支持使用transition属性添加过渡
    • 媒体查询也遵循层级关系，如果添加的层级不对，会影响效果是否能够实现

bootstrap框架

简介

  • 版本
    • 2.x：早期版本，轻度兼容移动端
    • 3.x：增强对移动端的兼容，常用
    • 4.x：测试版本，完全倾向移动端
  • 兼容性／支持
    • 完全依赖于 jquery，且 jquery 版本不能低于1.9.x
    • 可以兼容 ie8/9 及以后版本
  • 框架组件
    • 全局CSS样式
      • 包含一些预定义的表单，按钮，工具等，一般调用类名就能使用；
    • 组件
      • 包含一些进度条，导航栏，图标等，直接复制源代码修改即可
    • javascrip 插件
      • 包含编写好的轮播图，过渡效果，模态框等
    • 栅格系统
      • 通过行（row）与列（column）来布局，详见参考手册，栅格本质是浮动
  • 可定制
    • bootstrap支持详细的个性化定制，可以去掉不用的功能

使用方法

  • 需要使用页面模版，导入各种 js/css 等，参见 起步–>基本模版；
  • 基本嵌套结构
    • 页面区域
      • bootstrap容器（.container）
        • bootstrap行（.row）
          • 栅格系统（xs/sm/md/lg)
  • 具备两种容器 .container .container-fluid(100%)
  • 栅格嵌套
    • 栅格支持嵌套，不过应该在同一个 container 下在添加一个 row
    • 嵌套后的比例是父盒子的 n/12
  • 列偏移
    • 偏移的最大距离是包含自己在内的这一行的剩余份数
  • bootstrap 自带部分字体图标
    • 引用字体图标设置空的 span，并添加相应类名即可
  • 使用按钮同样是添加类名（btn btn-default）即可，不过类名（btn-xs）也可以控制大小；
  • 使用 bootstrap 提供的组件
    • 可以直接复制粘贴组件，去掉不要的标签元素
    • 组件复杂，嵌套关系多时，需要复制 bootstrap 的 css 源文件，替换最上层类名
    • 组件简单的话直接修改即可
  • 使用自定义字体图标时，可以仿照 bootstrap 的方式建立自己的字族文件；
    • 找到 bootstrap.css；复制 @font-face 定义，粘贴到通用 base 文件中
    • 替换 font-family 名称
    • 替换字体文件 URL；使用自定的字体文件URL
    • 在 base 中引入定义好的字体类名（一个字体有两个类名）
  • 媒体对象
    • 默认样式的媒体对象组件允许在一个内容块的左边或右边展示一个多媒体内容（图像、视频、音频）
  • 快速浮动
    • pull-left／pull-right
      • bootstrap 标准元素自带 清除浮动功能（container） 自己起的类名 就没有该功能
      • 不能应用于nav/media等；有专门的标签使用.navbar-left、navbar-right／media-left、media-right
  • 标签页
    • 通过锚链接的方式实现点击相应标签切换到相应页面内容
    • Togglable tabs，依赖于bootstrap的js组件实现；
    • 直接拷贝源代码使用，需要替换相应的类名；注意需要使nav与tab的id相同
  • banner用法；
    • 轮播图需要后台判断，用户请求时就轮播图会只请求一张
      • 复制源码 js插件 — Carousel
      • 删除不用的源码
      • 每张轮播图实际都有两张；大图设为除最小尺寸外显示，小图设为除最小尺寸外隐藏
      • 大图高度写死并设置中心缩放（backgroundposition：center／background-size：cover）
