移动端常用设置(1)

发表于 2018-01-18 | 分类于 Mobile Web

viewport属性

概念

如果把移动设备的浏览器(也有可能是app中的webview) ，当做相框的话，viewport就相当于相框中的画，可能会比相框小，可能会比相框大，如果刚好一样大，那就最好。

修改viewport

emmet写法：meta:vp

可选值

  • width — 设置layout viewport 的宽度，为一个正整数，或字符串”width-device”
  • initial-scale — 设置页面的初始缩放值，为一个数字，可以带小数
  • minimum-scale — 允许用户的最小缩放值，为一个数字，可以带小数
  • maximum-scale — 允许用户的最大缩放值，为一个数字，可以带小数
  • height — 设置layout viewport 的高度，很少使用
  • user-scalable — 是否允许用户进行缩放，值为”no”或”yes”, no 代表不允许，yes代表允许

关于缩放比例

缩放是相对于浏览器宽度来缩放的，缩放值越大，当前viewport的宽度就会越小，反之亦然。例如在iphone中，屏幕宽度是375，如果我们设置 initial-scale=0.5 ，此时viewport的宽度仍然为375，但是却需要750x1334的页面尺寸才能填满。

UI设计师通常会在设计时使用750x1334的尺寸，而js，css等获取的屏幕宽度都是375x667（涉及到dpr）

缩放的常用用法是根据设备的dpr，动态缩放页面的大小，来达到显示出1px线的问题，参考阿里的开源项目lib-Flexibl。

touch事件

tap事件

桌面端的click事件自带300ms延时，但是在移动端，添加了viewport属性后，延时会消失，所以为了兼容性，一般会需要封装一个简单的tap函数

zepto等移动端框架一般都会具备封装好的tap事件函数，可以拿来直接使用。

CSS媒体查询

使用 @media 查询，以针对不同的屏幕尺寸设置不同的样式，特别是需要设计响应式的页面，@media 是非常有用的。当你重置浏览器大小的过程中，页面也会根据浏览器的宽度和高度重新渲染页面。

  • 语法
  • 作用
    • 在媒体查询代码中添加判断代码；满足条件是改变外观；
    • 类似于if语句，支持使用transition属性添加过渡
    • 媒体查询也遵循层级关系，如果添加的层级不对，会影响效果是否能够实现
