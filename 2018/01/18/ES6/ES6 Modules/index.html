ES6 Modules

发表于 2018-01-18 | 分类于 JavaScript

ES6模块不是对象，而是通过 export 命令显示指定输出的代码，再通过 import 命令输入

  • 模块主要功能由 export 和 import 组成
    • export 命令用于规定模块对外的接口
    • import 命令用于导入其他模块提供的功能
  • 一个模块就是一个独立的文件，该文件内部所有变量，外部无法获取
    • 想要在外部获取，就必需使用 export 命令导出变量

严格模式

ES6 模块自动采用严格模式，所以不应在顶层代码使用 this

  • 严格模式的限制
    • 变量必须声明后再使用
    • 禁止 this 指向全局对象
    • 函数的参数不能有同名属性
    • 不能使用 with 语句
    • eval 不会再它的外层作用域引入变量
    • eval，arguments 不能重新赋值
    • arguments 不会自动反应函数参数的变化
    • 不能使用 arguments.callee，arguments.caller
    • 不能对只读属性赋值
    • 不能删除不可删除的属性
    • 不能使用前缀 0 表示 8进制

export

  • export 命令用于规定模块对外接口
  • 可以输出变量，函数以及类（class）
  • as关键字可以重命名导出的变量
  • 需要注意输出的格式
  • export 输出的接口，应该是动态更新的
  • export 应该位于模块顶层作用域

import

使用 export 命令定义了模块的对外接口以后，其他 JS 文件就可以通过 import 命令加载这个模块

  • 允许使用 as 关键字为导入的变量重命名
  • import 命令具备提升效果，直接提升到整个模块的头部
  • import 是静态执行的，不能使用表达式和变量进行引入
  • 可以使用 * 进行文件整体加载

export default

使用 export default 可以为模块指定默认输出，导入时不需要知道模块内部的名称，直接重命名即可

  • 指定默认输出后，导入时的 import 语句不需要使用大括号
  • 一个模块只能使用一次默认输出语句
  • export default 与 export 语句可以混合使用
  • 通过 export default 暴露的接口成员不能定义的同时直接暴露
