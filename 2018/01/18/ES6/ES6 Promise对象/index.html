ES6 Promise对象

发表于 2018-01-18 | 分类于 JavaScript

Promise 是异步编程的一种解决方案，比传统的解决方案——回调函数和事件——更合理和更强大。它由社区最早提出和实现，ES6 将其写进了语言标准，统一了用法，原生提供了 Promise对象。

概念

  • Promise 对象
    • Promise 是一个对象，从它可以获取异步操作的消息
    • 使用 Promise 对象，可以将异步操作以同步操作的方式表达出来，避免嵌套
  • Promise 对象状态
    • Pending — 进行中
    • Resolved — 已完成
    • Rejected — 已失败
  • Promise 对象状态特性
    • 对象状态不受外界影响
      • 只有异步操作的结果可以决定当前是哪一种状态
    • 对象状态一旦改变，就不会再次改变
      • 对象状态改变只用两种情况
        • Pending — Resolved
        • Pending — Rejected
  • 注意
    • 无法取消 promise对象，一旦创建就会立即执行，无法取消
    • 不设置回调函数，promise 内部抛出的出错误不会反应到外部
    • 当 promise对象处于 pending 状态时，无法获知目前进展到哪一个阶段

promise.prototype.then

Promise 实例具有 then方法，也就是说，then方法是定义在原型对象 Promise.prototype 上的。它的作用是为 Promise 实例添加状态改变时的回调函数。

then方法的第一个参数是 Resolved状态的回调函数，第二个参数（可选）是 Rejected状态的回调函数。

then方法返回的是一个新的 promise实例，这样即表明可以采用链式编程写法

promise.prototype.catch

Promise.prototype.catch 方法是 .then(null, rejection) 的别名，用于指定发生错误时的回调函数

  • 注意
    • 不推荐在 then方法里面定义 Rejected 状态的回调函数
    • Promise 对象的错误具有“冒泡”性质，会一直向后传递，直到被.catch()捕获为止
    • Promise 在 resolve语句后面，再抛出错误，不会被捕获，等于没有抛出
    • 如果没有使用 catch方法指定错误处理的回调函数，Promise 对象抛出的错误不会被接收
  • Node 监听未捕获的 reject错误
  • 返回的是 Promise对象，允许继续调用 then方法

Promise.all

Promise.all() 用于将多个 Promise实例包装成一个新的 Promise实例

  • p1, p2, p3都是 Promise实例，如果不是则先调用 Promise.resolve方法转换为 Promise实例
  • p的状态由p1, p2, p3 决定
  • 作为参数的 Promise 实例，自己定义了catch方法，那么它一旦被rejected，并不会触发 Promise.all() 的catch方法。

Promise.race()

Promise.race() 方法将多个 Promise 实例包装成一个新的 Promise 实例

Promise.resolve()

将一个对象转换为 Promise对象

根据参数不同，Promise.resolve() 具备4种转换结果

  • 参数是 Promise 实例
    • 直接返回这个实例
  • 参数是 thenabl对象（具备 then() 方法的对象）
    • 转换为 Promise 对象，并立即执行 thenable 对象的 then 方法
  • 参数不具有 then 方法或者不是对象
    • 返回一个 Promise 对象，状态为 Resolved
  • 不带任何参数
    • 直接返回一个 Resolved 状态的 Promise 对象
  • 需要注意执行顺序

Promise.reject()

该方法返回一个新的 Promise 实例，状态为 rejected

  • Promise.reject() 方法的参数，会作为 reject 的理由，变为后续方法的参数

自定义附加方法

done()

用于在回调链的尾端，保证抛出任何可能出现的错误

finally()

用于指定 Promise 对象都会执行的操作
