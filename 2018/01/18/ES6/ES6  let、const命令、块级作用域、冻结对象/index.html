ES6 let、const命令、块级作用域、冻结对象

发表于 2018-01-18 | 分类于 JavaScript

let 命令

ES6 新增变量声明方式，用法与 var 相同，但是只在当前代码块有效

特性

  • 不存在变量提升
  • 同作用域内禁止重复声明
  • 暂时性死区

应用

  • for 循环变量声明

const命令

ES6 新增常量声明方式，一旦声明，常量的值不能改变

特性

  • 声明后值不可变
  • 不能重复声明
  • 只在块级作用域有效
  • 不会进行常量提升
  • 暂时性死区
  • 一旦声明，必需立即赋值

应用

const 实际上是将变量指向的那个内存地址锁死。

简单类型的数据，值就保存在变量指向的那个内存地址，因此等同于常量。

复合类型的数据，变量指向的内存地址，保存的只是一个指针，const 能保证这个指针是固定的，但是可以改变该复杂类型的数据结构

块级作用域

  • ES6 之前的 JavaScript 不存在 块级作用域，会出现一些不合理的问题
  • let 实际上为 JavaScript 新增了块级作用域
  • 块级作用域与函数声明
    • 由于兼容性问题，应该避免在块级作用域内使用函数声明语句
    • 确实需要可以写成函数表达式的形式
