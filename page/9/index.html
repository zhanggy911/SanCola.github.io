Node 中的模块化

发表于 2018-01-18 | 分类于 NodeJS

模块化概念

模块系统

  • 由很多不同功能的模块文件组成的一个系统就是模块系统
  • node 里模块指的是实现不同功能的 JavaScript 文件
  • 模块之间相互依赖，相互通信，互相引用和调度完成一些列功能
  • 模块与模块之间不能有污染
  • 模块与模块之间通过特定的规则相互通信

require

  • 用来加载一个模块，该方法是同步加载执行的
  • 获取被加载模块中的 module.exports 接口对象
  • 被加载的文件模块后缀名可以省略
  • 模块是一个私有作用域，外部拿不到
  • 用户自己编写的模块，需要通过路径的形式进行加载

Node.js 概念、环境配置、基础

发表于 2018-01-18 | 分类于 NodeJS

概念

简单的说 Node.js 就是运行在服务端的 JavaScript
Node开发就是利用 ECMAscript + 第三方开源库 + Node平台环境API进行编程

Node 官网：https://nodejs.org/en/

  • Node.js 是一个构建于 Chrome V8 引擎之上的一个 JavaScript 执行环境
  • Node 采用了 Chrome 的 V8 引擎解析和执行 JavaScript 脚本代码
  • Node 使用 JavaScript 进行编程
  • Node 在基本的 JavaScript 语言之上，为 Ecmascript 提供了大量的底层编程接口
    • 文件IO
    • 网络IO
    • 处理数据
    • 操作数据库
    • 构建网络服务
  • Node.js 特征
    • 事件驱动（event-driven）
      • 在 Node 中，采用了类似于浏览器端的编程思想，事件模型编程思想
    • 非阻塞 IO 模型（non-blocking I/O model）
      • Node 大量使用类似于浏览器中 ajax 的方式进行编程
      • 绝大多数代码都是异步编程模型
